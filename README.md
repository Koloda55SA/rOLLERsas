# Rollerdrome

Android-приложение для автоматизации проката роликов на роллердроме.
Jetpack Compose + Cloud Firestore (без бэкенда). Простой интерфейс для кассиров.

## Возможности

- **Умный Канбан** (4 колонки): Свободны / Катаются / Заканчивается / Время ИСТЕКЛО.
  - Тап по свободному ролику → ввод номера бейджа → кнопка 30/60 → «СТАРТ».
  - Карточки обновляются каждую секунду и автоматически перемещаются между колонками.
  - Красная зона мигает, самые большие просрочки сверху; кнопка «Принять возврат» (доплата / простить).
- **Фоновый сервис** `TimerService` (foreground) держит таймеры в памяти даже при свёрнутом приложении.
- **Озвучка** при истечении времени: номер бейджа → общая фраза. Несколько одновременных — строго по очереди.
- **Фоновая музыка** играет всё время, приглушается до 20% на время озвучки, потом обратно 100%.
- **Запись голоса через UI**: в настройках экран «Озвучка» — запись общей фразы и номеров бейджей с микрофона, прослушивание и перезапись.
- **Скрытый экран бухгалтерии** (PIN, дефолт `7777`): расходы, смена (открытие/закрытие), аналитика, экспорт в PDF.
- **Админ-настройки**: кол-во роликов, сотрудниц, ЗП на сотрудницу, время открытия/закрытия, PIN.

## Модель данных (Firestore)

- `transactions` — сессии проката.
- `daily_expenses` — расходы за день (ID = дата).
- `shifts` — смены: время открытия/закрытия (ID = дата).

## Запуск (перед первым использованием)

1. **Firebase**: в консоли включить Cloud Firestore (Native). На время теста поставить открытые правила. Коллекции создавать не нужно — появятся автоматически.
2. **Голос (озвучка)**: открыть Настройки (шестерёнка, PIN 7777) → «Озвучка» → записать общую фразу и номера бейджей. Ничего в код класть не надо.
3. **Музыка (необязательно)**: положить `background_music.mp3` в `app/src/main/res/raw/` или запись `background_music` через UI. Без файла приложение работает без музыки.
4. **APK**: собирается в GitLab CI (`.gitlab-ci.yml`), артефакт `rollerdrome-debug-apk` на странице джоба.

## Локальная сборка

```
./gradlew assembleDebug
```

PDF-отчёты сохраняются в папку приложения и отправляются через «Поделиться».
